# Whisper — Локальная Транскрибация

[English](README.md) | [Русский](README.ru.md)

Инструмент командной строки для локальной транскрибации аудиофайлов с помощью [OpenAI Whisper](https://github.com/openai/whisper). После загрузки модели интернет не нужен — всё работает на вашей машине.

## Возможности

- Транскрибация любых аудиоформатов, поддерживаемых ffmpeg (mp3, wav, m4a, mp4, webm, ogg, …)
- Перевод аудио на английский (`--task translate`)
- Несколько форматов вывода: обычный текст, JSON, субтитры SRT, субтитры WebVTT
- Полный контроль над гиперпараметрами Whisper через аргументы командной строки
- Работает на CPU, видеокарте NVIDIA (CUDA) или Apple Silicon (MPS)

## Установка

```bash
# 1. Клонируйте или скачайте проект
git clone https://github.com/vetoshkina200220-glitch/whisper_transcribe_audio.git

# 2. Перейти в папку с проектом
cd transcribe_audio_whisper/

# 3. Создайте виртуальное окружение
python3 -m venv .venv
source .venv/bin/activate   # macOS / Linux
# .venv\Scripts\activate    # Windows

# 4. Установите зависимости
pip install -r requirements.txt

# 4. Проверьте наличие ffmpeg
ffmpeg -version
```

**Установка ffmpeg:**

- **macOS:** `brew install ffmpeg`
- **Linux (Debian/Ubuntu):** `sudo apt update && sudo apt install ffmpeg`
- **Linux (Fedora/RHEL):** `sudo dnf install ffmpeg`
- **Windows:** Скачайте сборку с [ffmpeg.org/download.html](https://ffmpeg.org/download.html), распакуйте архив и добавьте папку `bin/` в переменную среды `PATH`:
  1. Распакуйте архив в удобную папку, например `C:\ffmpeg`
  2. Откройте **Пуск** → найдите **«Изменение переменных среды»** → нажмите **Переменные среды**
  3. В разделе **Системные переменные** выберите **Path** → нажмите **Изменить** → **Создать**
  4. Введите `C:\ffmpeg\bin` и нажмите **ОК** во всех окнах
  5. Перезапустите открытые терминалы и проверьте: `ffmpeg -version`

## Быстрый старт

```bash
# Базовая транскрибация (автоопределение языка, модель base)
python transcribe.py audio.mp3

# Сохранить результат в файл
python transcribe.py audio.mp3 --output result

# Указать язык и более крупную модель
python transcribe.py audio.mp3 --model small --language ru

# Сгенерировать субтитры SRT
python transcribe.py audio.mp3 --output subtitles --output-format srt

# Перевести любой язык на английский
python transcribe.py audio.mp3 --task translate --output translated
```

## Сравнение моделей

| Модель | Параметры | VRAM | Скорость | Когда использовать |
|--------|----------|------|----------|--------------------|
| `tiny` | 39 М | ~1 ГБ | ~32x | Быстрые тесты, слабые машины |
| `tiny.en` | 39 М | ~1 ГБ | ~32x | То же, только английский — чуть точнее |
| `base` | 74 М | ~1 ГБ | ~16x | Хороший баланс для повседневных задач |
| `base.en` | 74 М | ~1 ГБ | ~16x | Только английский, немного точнее base |
| `small` | 244 М | ~2 ГБ | ~6x | Лучше для многоязычных задач |
| `small.en` | 244 М | ~2 ГБ | ~6x | Английский: качество близко к medium |
| `medium` | 769 М | ~5 ГБ | ~2x | Высокая точность, все языки |
| `medium.en` | 769 М | ~5 ГБ | ~2x | Лучшее качество для английского ниже large |
| `large` | 1550 М | ~10 ГБ | 1x | Максимальная точность, многоязычный |
| `large-v2` | 1550 М | ~10 ГБ | 1x | Улучшенный large |
| `large-v3` | 1550 М | ~10 ГБ | 1x | Наилучшая точность из всех |
| `turbo` | 798 М | ~6 ГБ | ~8x | Близко к large по качеству, намного быстрее |

**Рекомендация:** Начните с `base` для проверки. Используйте `small` или `medium` для рабочих задач. Используйте `large-v3`, когда точность критична и есть достаточно RAM/VRAM.

## Все аргументы командной строки

### Ввод / Вывод

| Аргумент | По умолчанию | Описание |
|----------|-------------|----------|
| `audio` | *(обязательный)* | Путь к аудиофайлу |
| `--output PATH` | stdout | Путь к выходному файлу без расширения |
| `--output-format` | `txt` | `txt` · `json` · `srt` · `vtt` |

### Модель и устройство

| Аргумент | По умолчанию | Описание |
|----------|-------------|----------|
| `--model` | `base` | Размер модели (см. таблицу выше) |
| `--device` | `cpu` | `cpu` · `cuda` · `mps` (при недоступности — откат на cpu) |

### Язык и задача

| Аргумент | По умолчанию | Описание |
|----------|-------------|----------|
| `--language КОД` | авто | Код языка: `ru`, `en`, `de`, `fr`, `ja`, … Ускоряет определение языка. |
| `--task` | `transcribe` | `transcribe` — оставить исходный язык; `translate` — вывод на английском |

### Гиперпараметры декодирования

| Аргумент | По умолчанию | Описание |
|----------|-------------|----------|
| `--temperature FLOAT` | `0.0` | Температура сэмплинга. 0 = жадное декодирование (детерминировано). Выше — больше вариативность, но точность может снизиться. |
| `--beam-size INT` | `5` | Ширина beam search. Выше = лучше качество, медленнее. Значение 1 — жадное декодирование. |
| `--best-of INT` | `5` | Количество кандидатов при `temperature > 0`. |

### Точность вычислений

| Аргумент | Описание |
|----------|----------|
| `--fp16` | Принудительно half-precision (fp16). Быстрее на GPU. |
| `--no-fp16` | Принудительно full-precision (fp32). Рекомендуется для CPU. |

> По умолчанию fp16 включается автоматически на CUDA; fp32 используется на CPU и MPS.

### Пороги качества

| Аргумент | По умолчанию | Описание |
|----------|-------------|----------|
| `--no-speech-threshold FLOAT` | `0.6` | Если вероятность «нет речи» превышает этот порог, сегмент пропускается. Меньше = строже фильтрация тишины. |
| `--compression-ratio-threshold FLOAT` | `2.4` | Сегменты с аномально высокой степенью сжатия считаются галлюцинациями и отбрасываются. |
| `--logprob-threshold FLOAT` | `-1.0` | Сегменты со средней log-вероятностью ниже порога отбрасываются. |

### Контекст и подсказки

| Аргумент | По умолчанию | Описание |
|----------|-------------|----------|
| `--condition-on-previous-text` / `--no-condition-on-previous-text` | включено | Передавать предыдущий сегмент как контекст. Отключите при зацикливании повторов. |
| `--initial-prompt ТЕКСТ` | нет | Текстовая подсказка для управления стилем и словарным запасом. Пример: `--initial-prompt "Медицинский разговор"` |

### Дополнительно

| Аргумент | По умолчанию | Описание |
|----------|-------------|----------|
| `--word-timestamps` | выкл | Добавить временные метки на уровне слов в JSON и форматы субтитров. |
| `--verbose` | выкл | Выводить каждый сегмент по мере транскрибации. При выключенном флаге вместо этого отображается прогресс-бар tqdm. |

## Прогресс-бар

По умолчанию во время транскрибации в консоли отображается прогресс-бар `tqdm`:

```
  8%|████▌                                                   | 1344/16000 [00:03<00:35, 415.06frames/s]
```

В нём видно:
- Обработанные кадры из общего числа
- Прошедшее время и расчётное время до завершения
- Скорость обработки в кадрах в секунду

Флаг `--verbose` переключает режим: вместо прогресс-бара будет выводиться текст каждого сегмента по мере транскрибации.

## Логирование

Каждый запуск пишет логи одновременно в консоль и в файл с датой в имени внутри директории `logs/`:

```
logs/
└── 2026-05-15.log
```

- Директория `logs/` создаётся автоматически при первом запуске.
- Каждый день получает свой файл (`YYYY-MM-DD.log`). Несколько запусков в один день **дописывают** в существующий файл — ничего не перезаписывается.
- В консоль выводятся сообщения уровня `INFO` и выше.
- В файл записывается всё начиная с уровня `DEBUG`, с полными временными метками.

Пример содержимого лог-файла:
```
2026-05-15 12:00:01,234 [INFO] Loading model 'base' on cpu...
2026-05-15 12:00:05,678 [INFO] Transcribing 'audio.mp3'...
2026-05-15 12:00:12,910 [INFO] Saved to result.txt
```

## Примеры форматов вывода

**txt** — обычный текст:
```
Привет, это пример транскрибации аудиофайла.
```

**json** — полный структурированный результат:
```json
{
  "text": "Привет, это пример...",
  "segments": [
    {"id": 0, "start": 0.0, "end": 3.5, "text": "Привет, это пример..."}
  ],
  "language": "ru"
}
```

**srt** — субтитры SubRip:
```
1
00:00:00,000 --> 00:00:03,500
Привет, это пример транскрибации.
```

**vtt** — субтитры WebVTT:
```
WEBVTT

00:00:00.000 --> 00:00:03.500
Привет, это пример транскрибации.
```

## Примеры использования

```bash
# Русское аудио, модель medium, сохранить как SRT
python transcribe.py interview.m4a --model medium --language ru --output interview --output-format srt

# Перевести французский подкаст в английский текст
python transcribe.py podcast.mp3 --task translate --output podcast_en

# Высокоточная транскрибация с временными метками слов
python transcribe.py lecture.wav --model large-v3 --word-timestamps --output lecture --output-format json

# Быстрая транскрибация для тестирования (модель tiny, жадное декодирование)
python transcribe.py draft.mp3 --model tiny --beam-size 1

# Ускорение на Apple Silicon
python transcribe.py audio.mp3 --model small --device mps
```
